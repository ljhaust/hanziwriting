# 汉字书写与古诗背诵产品规划与技术方案

本文件是平台的产品设计与技术方案权威说明，与代码实现保持一致。部署、环境变量与联调步骤以仓库根目录的 `README.md` 为准。

## 1. 产品目标

面向小学汉字书写与古诗背诵场景，提供**两个用户入口**：

- **管理后台**：供管理员、教师维护用户、汉字字库、古诗资源、作业任务并审阅练习记录。
- **原生微信小程序**：供学生完成今日作业、单字笔顺练习、古诗背诵填空并查看练习记录。

平台不再保留 PC 浏览器学习端或浏览器版小程序；学生侧只通过原生微信小程序使用。

核心目标：

- 让学生可以在微信小程序完成汉字笔顺学习、描红练习和古诗背诵。
- 让教师、管理员通过后台配置汉字、古诗、练习任务和用户。
- 所有业务数据集中存储于 MySQL，两端通过统一的 REST API 读写，前端不内置演示数据或失败兜底数据。

## 2. 用户角色

用户类型由 `user_account.user_type` 枚举区分，与代码 `PlatformService` 中的约定一致。

| 角色 | user_type | 主要入口 | 核心诉求 |
| --- | --- | --- | --- |
| 学生 | `student` | 微信小程序 | 练字、背诗、查看自己的练习记录与进度 |
| 家长 | `parent` | 微信小程序 | 陪同练习、查看记录（与小程序学生体验一致） |
| 教师 | `teacher` | 管理后台 | 配置作业任务、维护字库与古诗、查看完成情况 |
| 管理员 | `admin` | 管理后台 | 管理用户、字库、古诗、任务、记录与系统基础数据 |

> 管理后台登录接口仅放行 `admin` 与 `teacher` 两类账号（见 `PlatformService.authenticate`）。

## 3. 端侧规划

### 3.1 原生微信小程序端

定位：移动端日常练习入口，代码位于 `frontend/wechat-miniprogram`，单页承载五个底部页签。

| 页签 | 功能 |
| --- | --- |
| 首页 | 学生概况、连续练习天数、已练汉字/古诗统计、今日作业列表与完成度 |
| 练字 | 选择汉字、田字格描红书写、实时笔顺纠错、笔顺演示、提交评测生成记录 |
| 背诗 | 选择古诗、逐句展示、重点字手写填空、译文提示、完成后生成背诵记录 |
| 记录 | 当前学生练习记录列表，点击查看完成度、耗时与逐笔步骤详情 |
| 我的 | 头像、练习记录数、笔顺正确率、累计分钟、笔顺纠错开关 |

技术要点：

- 首屏调用 `GET /api/bootstrap` 一次拉取用户、字库、诗库、任务和记录，之后所有列表均来自该响应。
- 书写使用原生 2D Canvas；笔顺轨迹调用 `GET /api/hanzi/{characterText}/strokes` 从数据库获取，不请求第三方 CDN。
- 保存练习调用 `POST /api/records`，仅在接口成功后才把记录追加到列表，失败时不写入本地。
- 本地存储只保存笔顺纠错开关、演示速度等纯 UI 偏好。

### 3.2 管理后台

定位：内容、用户、任务和学习数据管理平台，代码位于 `frontend/src`，技术栈为 Vue 3 + Vite + Element Plus。

| 模块 | 功能 |
| --- | --- |
| 用户管理 | 列表、搜索、新增用户、切换启用/停用状态 |
| 汉字字库 | 列表、搜索、新增汉字（拼音、部首、笔画、年级）、切换推荐状态 |
| 古诗资源 | 列表、搜索、查看诗名/作者/朝代/年级/正文 |
| 作业任务 | 列表、搜索、发布任务（类型、投放对象、时间、状态、关联汉字与古诗）、删除 |
| 练习记录 | 列表、搜索、删除记录 |

管理后台登录使用 `POST /api/auth/login`，由后端比对数据库账号与 BCrypt 密码摘要；登录信息不写浏览器持久化存储，刷新需重新登录。

## 4. 后端技术方案

### 4.1 技术栈

| 层 | 选型 |
| --- | --- |
| 运行环境 | JDK 8 |
| 框架 | Spring Boot 2.7.18（`spring-boot-starter-web`、`spring-boot-starter-data-jpa`、`spring-boot-starter-validation`） |
| ORM | Spring Data JPA / Hibernate（`ddl-auto: update`，但生产以 `schema.sql` 为准） |
| 数据库 | MySQL 8.0+，字符集 `utf8mb4` |
| 密码摘要 | `spring-security-crypto` 的 `BCryptPasswordEncoder`（仅用于校验，未引入完整 Spring Security） |
| JSON 序列化 | Jackson，全局 `SNAKE_CASE` 命名策略 |
| 管理端 | Vue 3 + Vite + Element Plus + Vue Router |
| 小程序 | 原生微信小程序（不使用 uni-app / Taro） |

> 本项目**不使用 Redis、MyBatis、JWT、Swagger**。如后续需要引入，应同步更新本文件与 `pom.xml`。

### 4.2 工程分层

```text
backend/src/main/java/com/example/hanzi/
├── HanziWritingApplication.java   启动类
├── web/        PlatformController（REST）+ 异常映射
├── service/    PlatformService（业务与数据库读写）
├── domain/     JPA 实体（与数据库表一一对应）
├── repository/ Spring Data JPA 仓储
├── dto/        请求/响应 DTO（BootstrapResponse、HanziStrokeResponse 等）
└── config/     WebConfig（CORS 白名单）
```

控制器只做 HTTP 协议转换，业务规则集中在 `PlatformService`，实体不暴露敏感字段（如 `passwordHash` 标注 `@JsonIgnore`）。

## 5. 数据库设计

所有表结构以 `backend/src/main/resources/db/mysql/schema.sql` 为准，使用 `VARCHAR` 主键（业务前缀 + UUID，由服务端生成）。

### 5.1 核心表

| 表 | 说明 | 关键字段 |
| --- | --- | --- |
| `user_account` | 用户账号 | `id`、`username`(唯一)、`nickname`、`phone`、`user_type`、`status`、`join_date`、`password_hash` |
| `hanzi_character` | 汉字字库 | `id`、`character_text`(唯一)、`pinyin`、`radical`、`stroke_count`、`grade_level`、`recommended`、`strokes_json`、`medians_json`、`rad_strokes_json` |
| `poem` | 古诗资源 | `id`、`title`、`author`、`dynasty`、`content`、`annotation`、`translation`、`grade_level`、`textbook_version` |
| `practice_task` | 作业任务 | `id`、`task_name`、`task_type`、`target_type`、`target_id`、`start_time`、`end_time`、`status` |
| `practice_record` | 练习记录 | `id`、`user_id`、`user_name`、`task_id`、`task_name`、`item_type`、`item_id`、`item_name`、`complete_status`、`stroke_total`、`stroke_completed`、`mistake_count`、`hint_count`、`score_level`、`duration_seconds`、`practice_time` |

### 5.2 子表（一对多）

| 子表 | 归属 | 说明 |
| --- | --- | --- |
| `hanzi_tag` / `hanzi_stroke_desc` / `hanzi_compound` | `hanzi_character` | 标签、逐笔说明、组词 |
| `poem_keyword` | `poem` | 关键词 |
| `poem_sentence` | `poem` | 逐句拆分（`sort_no` 排序） |
| `poem_sentence_key_character` | `poem_sentence` | 背诵填空重点字 |
| `task_item` | `practice_task` | 任务包含的汉字/古诗条目（`sort_no` 排序） |

子表均通过外键 `ON DELETE CASCADE` 跟随主表删除；JPA 实体以 `@ElementCollection` 或 `@OneToMany(cascade = ALL, orphanRemoval = true)` 映射。

### 5.3 笔顺轨迹三列

`hanzi_character` 表的笔顺功能要求同时配置三列，缺一不可：

- `strokes_json`：Hanzi Writer SVG 笔画路径 JSON 数组；
- `medians_json`：与笔画顺序对应的中线坐标 JSON 数组；
- `rad_strokes_json`：部首笔画索引 JSON 数组。

三列必须为合法 JSON 数组，且 `strokes_json` 与 `medians_json` 笔画数量及顺序一致。否则 `GET /api/hanzi/{text}/strokes` 返回 404 或 500，前端不会请求 CDN 或构造兜底轨迹。应通过受控的数据导入脚本写入，不要把轨迹硬编码到小程序。

## 6. REST API 规划

所有接口前缀 `/api`，请求/响应为 JSON，命名遵循 `snake_case`。删除成功返回 HTTP 204。完整清单见 `README.md` 的「REST API 清单」。

| 方法 | 路径 | 调用端 | 说明 |
| --- | --- | --- | --- |
| `GET` | `/api/bootstrap` | 两端 | 聚合返回用户、字库、诗库、任务、记录 |
| `POST` | `/api/auth/login` | 管理端 | 账号 + BCrypt 摘要校验 |
| `GET`/`POST` `/api/users`、`PUT /api/users/{id}/status` | 管理端 | 用户查询、创建、启停 |
| `GET`/`POST` `/api/hanzi`、`PUT /api/hanzi/{id}/recommended` | 管理端 | 字库查询、创建、推荐切换 |
| `GET` | `/api/hanzi/{characterText}/strokes` | 小程序 | 数据库笔顺轨迹 |
| `GET` | `/api/poems` | 管理端 | 古诗查询 |
| `GET`/`POST` `/api/tasks`、`DELETE /api/tasks/{id}` | 管理端 | 任务查询、发布、删除 |
| `GET`/`POST` `/api/records`、`DELETE /api/records/{id}` | 两端 | 记录查询、提交、删除 |

设计原则：

- 启动聚合接口 `/api/bootstrap` 减少首屏接口往返；写操作返回已持久化的完整实体，前端直接以其为真值更新列表。
- 资源不存在统一抛 `ResourceNotFoundException` → HTTP 404；登录失败抛 `InvalidCredentialsException` → HTTP 401，且不区分账号或密码错误。
- 校验由 `javax.validation` 注解（`@NotBlank`、`@NotNull`、`@Pattern`）在控制器入口完成。

## 7. 数据来源约束

- 两端的用户、汉字、古诗、任务、记录、笔顺轨迹**均以后台接口响应为唯一业务数据来源**。
- 前端**不包含** mock 集合、失败本地兜底或本地业务记录；接口失败显示错误并提供重试，不静默返回样例数据。
- 笔顺数据从数据库读取，小程序不直接请求第三方 CDN。
- 本地存储只允许保存纯 UI 偏好（笔顺纠错开关、演示速度）。
- 仓库不提交数据库密码、令牌、私钥或 `.env.local`；凭据只通过环境变量注入。

## 8. 后续可选演进

以下能力当前**未实现**，列入未来演进，引入时须同步更新本文档与代码：

- 微信登录（`wx.login` + `openid` 绑定），替代当前以数据库学生 ID 关联记录的方式。
- 班级与教师布置作业的完整闭环（`target_type=class` 的语义化落地）。
- 学习报告导出、错题本、教材版本完整适配。
- 接口鉴权（Token / Session），目前管理后台登录仅做一次校验，不维护会话。
- 笔顺轨迹的后台可视化导入工具（当前需手工写入三列 JSON）。
