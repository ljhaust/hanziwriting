# 汉字书写与古诗背诵产品规划与技术方案

## 1. 产品目标

本产品从单一“汉字笔顺演示工具”升级为面向学生、家长、老师和管理员的书写学习平台，覆盖微信小程序、PC 学习端和后台管理端。

核心目标：

- 让学生可以在微信小程序和 PC 端完成汉字笔顺学习、描红练习和古诗词书写背诵。
- 让家长或老师可以通过后台配置汉字、古诗词、练习任务和学习内容。
- 让管理员可以维护用户、内容、字库、诗词库、学习记录和系统基础数据。
- 后端采用 Java 8、Spring Boot、MySQL，提供统一 API 支撑小程序、PC 端和后台管理端。

## 2. 用户角色

### 2.1 学生

主要使用微信小程序和 PC 学习端。

核心诉求：

- 搜索并练习单个汉字。
- 按今日任务连续练习多个汉字。
- 书写古诗词中的字，辅助背诵。
- 查看自己的练习进度和完成反馈。

### 2.2 家长

主要使用微信小程序或 PC 学习端。

核心诉求：

- 给孩子选择练习字或古诗词。
- 查看孩子练习记录。
- 快速了解哪些字写错较多、哪些诗词未掌握。

### 2.3 老师

主要使用 PC 端和后台管理端。

核心诉求：

- 配置班级练习任务。
- 维护课本生字和古诗词内容。
- 查看学生完成情况。

### 2.4 管理员

主要使用后台管理端。

核心诉求：

- 管理用户、角色、权限。
- 管理汉字、字组、古诗词、年级、课程。
- 查看系统数据和学习数据。

## 3. 端侧规划

## 3.1 微信小程序端

定位：移动优先的练习入口，适合学生日常短时高频使用。

### 3.1.1 首页

功能：

- 展示今日练习任务。
- 展示最近练习的汉字和古诗词。
- 提供搜索入口。
- 提供“汉字练习”“古诗词练习”“我的记录”入口。

首屏重点：

- 今日任务进度，例如 `3/5`。
- 当前推荐练习内容。
- 一个明确主按钮，例如“继续练习”。

### 3.1.2 汉字练习

功能：

- 搜索单个汉字。
- 查看笔顺动画。
- 进入描红练习。
- 完成后显示反馈。
- 支持下一个字、再练一次。

反馈指标：

- 完成笔画数。
- 提示次数。
- 是否按正确笔顺完成。
- 练习评级。

### 3.1.3 古诗词练习

功能：

- 按年级、作者、关键词搜索古诗词。
- 查看古诗全文。
- 逐句练习书写。
- 支持“隐藏部分文字”进行背诵。
- 点击诗句中的字进入单字笔顺练习。
- 完成后生成诗词练习记录。

推荐交互：

- 第一阶段：展示全文，用户按字书写。
- 第二阶段：隐藏重点字，用户补写。
- 第三阶段：隐藏整句，用户默写。

### 3.1.4 我的记录

功能：

- 查看汉字练习记录。
- 查看古诗词背诵记录。
- 查看连续练习天数。
- 查看易错字。

## 3.2 PC 学习端

定位：适合大屏书写、课堂投屏、家长陪练。

功能：

- 当前已有 Vue 3 + Vite 页面继续演进为 PC 学习端。
- 支持汉字搜索和练习。
- 支持今日字组练习。
- 支持古诗词书写练习。
- 支持扫码在小程序继续练习。

与小程序差异：

- PC 端更适合展示大田字格和完整练习报告。
- 小程序端更适合碎片化练习。

## 3.3 后台管理端

定位：内容、用户、任务和学习数据管理平台。

推荐技术：

- 前端：Vue 3 + Vite + Element Plus。
- 后端：Java 8 + Spring Boot。
- 数据库：MySQL。

### 3.3.1 用户管理

功能：

- 用户列表。
- 新增、编辑、禁用、删除用户。
- 按用户名、手机号、角色、状态搜索。
- 重置密码。
- 分配角色。

用户类型：

- 管理员。
- 老师。
- 家长。
- 学生。

### 3.3.2 角色权限管理

功能：

- 角色列表。
- 角色授权。
- 菜单权限。
- 接口权限。

基础角色：

- 超级管理员。
- 内容管理员。
- 老师。
- 普通用户。

### 3.3.3 汉字管理

功能：

- 汉字列表。
- 搜索汉字。
- 配置拼音、部首、笔画数、年级、标签。
- 查看字形数据来源。
- 配置推荐字组。

说明：

笔顺动画仍可使用 Hanzi Writer 数据能力，后台主要维护业务属性，例如年级、课程、标签、是否推荐。

### 3.3.4 古诗词管理

功能：

- 古诗词列表。
- 新增、编辑、删除古诗词。
- 配置标题、作者、朝代、正文、注释、译文、年级、教材版本。
- 配置重点字、易错字、背诵等级。
- 支持诗句拆分。

### 3.3.5 练习任务管理

功能：

- 新建练习任务。
- 选择任务类型：汉字任务、古诗词任务、混合任务。
- 指定练习对象：全部学生、班级、个人。
- 设置开始时间、截止时间。
- 查看任务完成情况。

### 3.3.6 学习记录管理

功能：

- 查看学生练习记录。
- 查看汉字完成情况。
- 查看古诗词背诵情况。
- 查看提示次数、错误次数、完成时间。
- 导出学习记录。

## 4. 后端技术方案

## 4.1 技术栈

- JDK：Java 8。
- 框架：Spring Boot 2.x。
- ORM：MyBatis 或 MyBatis-Plus。
- 数据库：MySQL 5.7/8.0。
- 权限：Spring Security + JWT，或 Sa-Token。
- 缓存：Redis，可在 V1.1 后引入。
- 文件存储：本地存储起步，后续可扩展对象存储。
- API 文档：Swagger / Knife4j。

## 4.2 后端模块

建议模块：

- `user`：用户、角色、权限。
- `content`：汉字、古诗词、字组、标签。
- `practice`：练习任务、练习记录、完成结果。
- `auth`：登录、鉴权、微信登录绑定。
- `system`：字典、配置、日志。

## 6. API 接口规划

## 6.1 认证接口

- `POST /api/auth/login`：后台账号登录。
- `POST /api/auth/wx-login`：微信小程序登录。
- `GET /api/auth/profile`：获取当前用户信息。
- `POST /api/auth/logout`：退出登录。

## 6.2 用户管理接口

- `GET /api/admin/users`：用户分页查询。
- `POST /api/admin/users`：新增用户。
- `PUT /api/admin/users/{id}`：编辑用户。
- `DELETE /api/admin/users/{id}`：删除用户。
- `PUT /api/admin/users/{id}/status`：启用/禁用用户。
- `PUT /api/admin/users/{id}/password`：重置密码。

## 6.3 汉字接口

- `GET /api/hanzi/search?keyword=点`：搜索汉字。
- `GET /api/hanzi/{id}`：获取汉字详情。
- `GET /api/hanzi/recommended`：获取推荐字组。
- `POST /api/admin/hanzi`：后台新增汉字业务属性。
- `PUT /api/admin/hanzi/{id}`：后台编辑汉字业务属性。

## 6.4 古诗词接口

- `GET /api/poems`：古诗词分页查询。
- `GET /api/poems/{id}`：古诗词详情。
- `GET /api/poems/{id}/sentences`：古诗词句子列表。
- `POST /api/admin/poems`：后台新增古诗词。
- `PUT /api/admin/poems/{id}`：后台编辑古诗词。
- `DELETE /api/admin/poems/{id}`：后台删除古诗词。

## 6.5 练习接口

- `GET /api/practice/today`：获取今日练习。
- `POST /api/practice/records`：提交练习记录。
- `GET /api/practice/records/my`：查询我的练习记录。
- `GET /api/admin/practice/records`：后台查询练习记录。

## 7. 古诗词书写功能设计

## 7.1 学习模式

古诗词练习建议分三种模式：

### 模式一：看诗写字

展示完整古诗，用户点击某个字进入单字笔顺和描红。

适合初学。

### 模式二：重点字补写

系统隐藏诗句中的重点字，学生需要按顺序书写。

适合巩固易错字。

### 模式三：整句默写

隐藏整句或整首诗，学生按字逐个书写。

适合背诵检测。

## 7.2 页面流程

流程：

1. 选择古诗词。
2. 查看标题、作者、全文。
3. 选择练习模式。
4. 按句进入书写。
5. 每个字完成后自动跳到下一个字。
6. 整句完成后显示句子完成反馈。
7. 全诗完成后显示背诵结果。

## 8. MVP 范围建议

不要一开始同时做完整班级、权限、报告和全部诗词能力。建议 MVP 聚焦最小闭环。

## 8.1 MVP 必做

- 微信小程序：汉字搜索、单字书写、今日练习、古诗词列表、古诗词逐字书写。
- PC 端：沿用当前 Vue 页面，增加古诗词练习入口。
- 后台管理端：用户管理、汉字管理、古诗词管理。
- 后端：登录、用户、汉字、古诗词、练习记录基础接口。
- MySQL：用户、汉字、古诗词、练习记录基础表。

## 8.2 MVP 暂缓

- 班级管理。
- 老师布置任务。
- 复杂权限矩阵。
- 学习报告导出。
- Redis。
- 文件上传。
- 多教材版本完整适配。

## 10. 工程结构建议

建议拆为多项目目录：

```text
hanzi-writing-platform/
  backend/
    hanzi-server/              # Java 8 + Spring Boot + MySQL
  frontend/
    pc-web/                    # 当前 Vue 3 + Vite PC 学习端
    admin-web/                 # Vue 3 + Vite + Element Plus 后台管理端
  miniprogram/
    wechat-app/                # 微信小程序端
  docs/
    prd/
    api/
    database/
```

当前项目可以先作为 `frontend/pc-web` 的基础迁移。

## 11. 关键决策点

后续开发前需要确认：

1. 小程序是否必须使用原生微信小程序，还是可以使用 uni-app/Taro。
2. 后台管理端是否独立登录，还是与小程序用户体系统一。
3. 用户是否需要区分学生、家长、老师绑定关系。
4. 古诗词数据来源是否手工维护，还是需要导入现有资料。
5. 是否需要支持班级和老师布置作业。
6. 是否需要上线部署，部署环境是本机、云服务器还是内网服务器。
